Using a map with your Claude

The maps run on their own in any browser — spinning, replay, search, the suggestion engine, and import/export all work offline, no account needed. But the real power is the map and a Claude working as a pair: the map finds where a leap wants to happen; Claude proposes what it might be. Here's how to wire them together. Copy a prompt, paste it into any Claude chat, then drop Claude's answer into the map's Import box.

Quick orientation: every map has a Share a sequence card. Export gives you a small block of JSON describing your whole map; Import rebuilds a map from such a block. That JSON is the language you and Claude pass back and forth.

1 · Ask Claude to build you a starter map

Gives you a whole sequence to paste into Import and start spinning.

Help me build a "Gel Brain" sequence — a map of how a line of thinking unfolds.
Output ONLY a JSON object in exactly this format:

{"format":"gelbrain.v1","thoughts":[
  {"t":"","v":[grounded,tension,relation,mechanism,scale,ethic],"p":}
]}

Each axis is 0.00–1.00:
 grounded  (0 vision … 1 established fact)
 tension   (0 no problem … 1 the hard edge / contradiction)
 relation  (0 solitary/technical … 1 cooperation, bonding)
 mechanism (0 abstract … 1 concrete how-it-works)
 scale     (0 molecular … 1 civilizational)
 ethic     (0 tool/extraction … 1 companion/endowment)

Order the thoughts the way they'd actually arise. Every "p" must be a SMALLER
index than its own thought (parents come first; the first thought is -1).
Give 15–25 thoughts that move from a seed, through grounding, to a hard edge,
a reframe, and a scaling step.

Topic: 
Output ONLY the JSON, no prose.

2 · Ask Claude to find and fill an unmade leap

Paste your exported map in; Claude returns the whole sequence with new bridge thoughts appended, ready to Import.

Here is my Gel Brain sequence. "t" is each thought; "p" is the index of the
earlier thought it grew from.



Read it as a line of reasoning. Find two thoughts that rhyme in spirit (both
high on relation or ethic) but sit far apart with no link between them — the
unmade leap. Then write 1–3 NEW bridge thoughts that connect them.

Return the FULL sequence: my original thoughts with their "t","v","p" unchanged,
plus your new bridge thoughts appended at the end, each "p" pointing to an
existing thought's index. Same axis meanings as before. Output ONLY the
gelbrain.v1 JSON.

3 · Ask Claude where the thinking is bold — or stalling

No JSON back, just a read on the shape of the path.

Here's my Gel Brain sequence — "t" is each thought, "p" is the thought it grew
from:



Tell me three things: where did my thinking make its boldest leap, where does it
look like I stayed too safe or stalled, and what is one audacious direction the
path is pointing at that I haven't taken yet?

4 · Ask Claude to place a single new thought

For adding one node by hand: Claude gives you the six numbers, which you can drop into your exported JSON as a new thought's v, then re-Import.

Using the Gel Brain six axes — grounded, tension, relation, mechanism, scale,
ethic (each 0.00–1.00; grounded = vision→fact, tension = the hard edge,
relation = cooperation/bonding, mechanism = how-it-works, scale =
molecular→civilizational, ethic = tool→companion) — score this single thought.
Reply with ONLY a six-number JSON array, e.g. [0.5,0.2,0.8,0.3,0.7,0.9].

Thought: ""

The shortcut: build it live, inside Claude

If you open a map inside a Claude conversation, the map's own Add a thought box places new thoughts by true meaning automatically — Claude reads each one and sets it on the six axes for you, no copy-paste. Outside Claude, that one box falls back to approximate placement; everything else still works perfectly. So: build live in Claude, then Export and share the frozen file anywhere.

Sharing with your group

  • Send a link, not a file. If GitHub Pages is on, just share the map's URL and people click to spin it — nothing to download.
  • Send the file. Or hand them the .html (the repo's green Code → Download ZIP); they double-click it and it opens working, offline.
  • Send a sequence. Export your map to JSON and paste it into a thread — anyone can Import it, press Replay, and build onto the end.
